Incident Reporting System (IRS)

Architecture and Data Processing (CSV, XML, JSON)

IRS Data Flow Diagram

The Incident Reporting System (IRS) is a tool designed to standardize and streamline the notification and management of security incidents, fundamental for small businesses.

Data Acquisition and Persistence: The core of the system starts with a form to collect incident data, ensuring correct tracking. Initial data is stored and then exported to a CSV file (incidencias.csv); the following data are test data.

Format Migration and Validation (XML): The system includes functionality for the transformation of data from CSV to XML format (incid√®ncies.xml), which is crucial for information exchange. To verify the resulting structure, a browser (like Firefox, Chrome) and code formatting tools are used.

Processing and Analysis (Python): A small Python program was developed to read incidents from the XML file and process the information to obtain meaningful statistics. Although it is a console application, visual format (colors, tabs) is prioritized to facilitate reading the obtained information. This phase also includes logic to filter out data that does not provide useful information or is erroneous (e.g., an incident with a future date).

Final Storage (JSON): The processed and analyzed information is stored in a JSON file (incidencies.json). Before saving, the system checks if the file already exists. If it exists, the user can choose to Overwrite the content or Append the new incidents (avoiding ID duplicates). The manipulation of these data formats (JSON, XML) is aligned with knowledge of client scripting languages and information management for data exchange.

Project Goal: Efficient, structured security incident handling across multiple data formats (CSV, XML, JSON).
